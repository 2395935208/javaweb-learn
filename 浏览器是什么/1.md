# 浏览器如何运行
浏览器发送请求，服务器接收请求，获取资源，生成反馈传递回到服务端
## 浏览器多进程是什么
浏览器把不同的任务拆分成多个相互隔离的进程，让不同任务并行执行，提高安全性和稳定性
以chrome为例，主要包含：
- Browser Process（浏览器的主进程）
浏览器的大脑，地址栏UI，进程调度等
- Renderer Process（渲染进程）
html解析（DOM），css解析（CSSOM），js执行，页面布局
- Network Process（网络进程）
专门处理：HTTP/HTTPS请求，DNS查询，连接管理（TCP/TLS）
- GPU Proess（图形进程）
负责页面绘制加速，css动画，视频渲染
## 浏览器输入url发生了什么
输入https://www.baidu.com以后，浏览器会进行如下步骤：
1.解析url，拆成：
协议：http
域名：www.baidu.com
路径：/
参数：？xxx
2.检查缓存
能不请求服务器就不请求
三种缓存分别是：
Memory Cache（内存缓存），存在内存中，最快速但是断开页面就没有了
Disk Cache（磁盘缓存），存在硬盘，速度比内存慢，可以长期保存
Service Worker Cache（离线缓存）
若果命中缓存了，甚至可以直接返回页面

## 建立TCP连接
网络传输本身是不可靠的可能：丢包，顺序错乱
tcp的作用就是保证：数据不丢，数据按顺序到达，出错会重传
三次握手：
1. 客户端 → 服务器：SYN（我要连接）
2. 服务器 → 客户端：SYN + ACK（同意 + 确认）
3. 客户端 → 服务器：ACK（确认建立）
连接建立成功后才能传输数据
HTTPS多出来的步骤是：TLS握手
验证服务器身份（证书），协商加密方式，生成对称密钥（重点）
