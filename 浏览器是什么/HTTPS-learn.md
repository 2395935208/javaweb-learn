# HTTPS是什么
HTTPS就是HTTP+加密，本质还是HTTP，只是在HTTP和TCP之间加了一层TLS/SSL，保证数据的安全性
## 为什么需要HTTPS
HTTP是明文传输，数据不安全，HTTPS可以解决三个问题：
- 机密性：内容加密，防止被窃听（比如你的密码、Token 不会被截获看到）
- 完整性：防止数据被篡改（比如运营商劫持给你插广告）
- 身份认证：确认你连的服务器就是真的那个服务器，不是钓鱼网站
## **TLS握手过程（重点）**
简化版流程：

- 1.ClientHello：客户端发起，说明支持的加密算法、TLS版本
- 2.ServerHello + 证书：服务器返回证书（包含公钥），客户端验证证书是否可信（CA 签发）
- 3.密钥交换：双方通过非对称加密（如 RSA 或更常见的 ECDHE）协商出一个对称密钥
- 4.切换到对称加密通信：后续所有数据用这个对称密钥加密传输
## 关键区别（前者为HTTP，后者为HTTPS）
- 默认端口：80，443
- 是否加密：否，是（TLS）
- 证书：不需要，需要CA签发的证书
- 性能：快，握手有额外开销
